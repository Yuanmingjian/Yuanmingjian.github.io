20 js的递归中给结果数组加入 要用current .slice() 而不能用current

在使用回溯算法进行组合或排列时，我们通常需要创建和操作当前组合或排列的数组。由于 JavaScript 中数组是引用类型，这意味着在进行组合或排列过程中，如果直接对当前组合或排列的数组进行操作，可能会引起其他已经存储的组合或排列的数组的变化，导致结果不正确。

因此，为了避免这种问题，我们需要创建当前组合或排列的数组的副本，以便对副本进行操作，而不会影响到原始数组。而 arr.slice() 可以返回原数组的一个副本，因此常常被用来创建一个新的数组，以便进行回溯过程中的操作，而不影响之前存储的，已经完成的组合或排列的结果。

例如，在找到一个由输入数组 arr 中所有顺序相关元素的组合时，我们通常会定义一个回溯函数，将当前组合作为参数传入函数，并在函数中进行组合或排列。在这种情况下，我们需要在回溯函数中使用的新组合数组，由于需要作为参数传入函数，因此不能直接拷贝当前组合数组（因为这将使当前组合的任何变化都反映在已经存储的组合数组上），而是应该使用 arr.slice() 创建一个新数组，并对新数组进行操作。这样，对新数组的所有更改都不会反映在已经存储的组合数组中，保证了最终的结果的正确性。

举例：

```javascript
function findAllCombinations(arr, start, end, current, result) {
if (current.length > 0) {
result.push(current.slice()); // 将当前组合加入结果
}

for (let i = start; i <= end; i++) {
current.push(arr[i]);
findAllCombinations(arr, i + 1, end, current.slice(), result); // 递归到下一个元素，并使用当前组合的副本
current.pop();
}
}

const arr = [1, 2, 3];
const result = [];

findAllCombinations(arr, 0, arr.length - 1, [], result);

console.log(result); // 输出 [[], [1], [1, 2], [1, 2, 3], [1, 3], [2], [2, 3], [3]]
```

在上面的代码中，我们定义了一个查找 arr 数组中所有组合的函数 findAllCombinations。函数定义了5个参数，其中两个是结果变量，其他三个是对组合进行操作的参数。

在函数的核心循环中，我们将 current 数组的拷贝传递给函数，以避免在使用 current 进行操作时更改原始组合数组。比如，我们递归调用 findAllCombinations 函数时传递的组合数组就要使用 current.slice() 而不是 current。 最后，函数将所有组合存储在 result 数组中，并在函数结束后返回该数组。

这样，就可以确保不会因为直接更改已经存储的组合数据而导致寻找不完整或重复的组合。

不拷贝arr的副本 直接将数组加入结果中发生的影响

如果您不使用 arr.slice()，而是直接将 arr 作为当前组合进行操作并将其添加到结果数组中，可能会对结果产生意外的影响。我们可以看一下下面的示例代码：

```javascript
function findAllCombinations(arr, start, end, current, result) {
if (current.length > 0) {
result.push(current); // 将当前组合加入结果，这里不使用 arr.slice()
}

for (let i = start; i <= end; i++) {
current.push(arr[i]);
findAllCombinations(arr, i + 1, end, current, result);
current.pop();
}
}

const arr = [1, 2, 3];
const result = [];

findAllCombinations(arr, 0, arr.length - 1, [], result);

console.log(result); // 输出 [[], [1], [1, 2], [1, 2, 3], [1, 2], [1, 3], [1, 3], [2], [2, 3], [2], [3], [3]]
```

在上面的代码中，我们定义了一个查找 arr 数组中所有组合的函数 findAllCombinations，其中对于已经生成的组合数组没有使用 arr.slice()。这意味着在这里我们直接将 current 数组添加到结果数组中，而不是添加它的拷贝。由于数组是引用类型，这可能导致我们改变 current 数组进而产生一些奇怪的结果。

输出结果表明，在组合中添加一个元素之后，组合数组的其他部分也被莫名其妙地更改了。这是因为 current 数组直接被添加到了结果数组中，而不是 arr.slice() 的拷贝，所以添加新元素时，也添加并更改了已经存储在结果数组中的组合数组。

在使用回溯算法时，为了避免出现这种错误，通常会使用 arr.slice() 创建一个当前组合的副本，以确保在添加新元素时不会改变原始组合数据。
